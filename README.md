## 🛠️ 当前环境概览

为了保证作业的复现和随时随地的便捷访问，当前您正处于一个定制化的云端开发环境中：

* **操作系统基底**：基于 **Artix Linux** 根文件系统构建的 Docker 容器。
* **图形界面**：运行轻量级的 **XFCE** 桌面环境，并通过 **noVNC** 将画面串流到您的浏览器中。
* **自动化工作流**：每次启动该环境时，系统已自动同步了最新的作业仓库 (`https://github.com/CodingXiaoheng/homework_hznu`)，并为您自动启动了当前的 VS Code。

您可以像使用本地电脑一样，在这个网页端的完整 Linux 桌面中编写、调试和运行代码。

> ### ⚠️ 重要警告：本沙盒无持久化存储！
> 
> 
> 请务必注意，当前您所在的 Docker 容器环境是**临时且非持久化**的。
> * **一旦容器重启或关闭，所有本地修改都会灰飞烟灭！**
> * 如果您在此环境中修改了代码、运行了实验或编写了新作业，**请务必使用 Git 提交并推送到远程仓库 (`git commit` & `git push`)**，或者手动将文件复制下载到您的本地电脑。

## 🚀 如何运行与复现代码

### 1. 常规代码

对于一般的作业代码，您只需在 VS Code 中打开对应的文件，使用终端（`Ctrl + ~` 或 `Terminal -> New Terminal`）进入相应目录，使用对应的编译器或解释器运行即可（如 `python main.py` 或 `gcc main.c -o main && ./main`）。

### 2. ❄️ 使用 Nix 的高可复现环境

为了彻底解决“在我的电脑上能跑”的环境依赖问题，部分作业使用了 **Nix 包管理器**。

* **如何识别**：如果某个作业的目录下包含 `shell.nix` 文件，说明该作业依赖 Nix 构建隔离环境。
* **如何运行（全自动）**：**您不需要手动配置任何东西！** 相关的运行脚本（如 `.sh` 或 `.py`）开头已经内置了检测逻辑。当您直接运行脚本时，它会检查当前是否处于 `nix-shell` 环境中；如果不在，脚本会**自动在 `nix-shell` 中重启自身**并下载所需的依赖。
