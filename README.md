## 🛠️ 当前环境概览

为了保证作业的复现和随时随地的便捷访问，当前您正处于一个专属的**家里云（Home Lab）虚拟机**环境中：

* **系统架构**：基于 **Arch Linux** 构建的虚拟机（VM）。
* **图形界面**：运行轻量级的 **XFCE** 桌面环境，让您可以像使用本地电脑一样，顺畅地进行远程交互与代码编写。

> ### ⚠️ 重要须知：代码重置与环境纯净度
> 
> 
> 请务必了解当前虚拟机的存储机制与使用规范：
> * **🔄 代码仓库强制覆盖**：每次虚拟机重启时，系统会**强制从 GitHub 拉取最新代码并覆盖**当前的作业仓库 (`https://github.com/CodingXiaoheng/homework_hznu`)。因此，如果您在此环境中修改了代码或编写了新作业，**请务必在重启前使用 Git 提交并推送到远程仓库 (`git commit` & `git push`)**。任何未推送的仓库内更改都会在下次重启时灰飞烟灭！
> * **💾 局部持久化存储**：与临时的沙盒不同，虚拟机中的**其他位置是持久化存储的**。除了被强制覆盖的作业仓库目录外，您存放在系统其他位置的文件不会随重启而丢失。
> * **🚫 请勿随意安装软件**：为了保证作业运行环境的纯净性与高可复现性，**请勿在系统内随意使用包管理器（如 `pacman`）安装全局软件**。特定的环境依赖请参考下方使用 `Nix` 解决。

## 🚀 如何运行与复现代码

### 1. 常规代码

对于一般的作业代码，您只需在 VS Code 中打开对应的文件，使用终端（`Ctrl + ~` 或 `Terminal -> New Terminal`）进入相应目录，使用对应的编译器或解释器运行即可（如 `python main.py` 或 `gcc main.c -o main && ./main`）。

### 2. ❄️ 使用 Nix 的高可复现环境

为了彻底解决“在我的电脑上能跑”的环境依赖问题，同时**避免在虚拟机中全局安装脏依赖**，部分作业使用了 **Nix 包管理器**。

* **如何识别**：如果某个作业的目录下包含 `shell.nix` 文件，说明该作业依赖 Nix 构建隔离环境。
* **如何运行（全自动）**：**您不需要手动配置任何东西！** 相关的运行脚本（如 `.sh` 或 `.py`）开头已经内置了检测逻辑。当您直接运行脚本时，它会检查当前是否处于 `nix-shell` 环境中；如果不在，脚本会**自动在 `nix-shell` 中重启自身**并下载所需的依赖，用完即焚，绝不污染当前系统。
